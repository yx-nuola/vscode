VS Code 插件日志面板功能开发与性能优化总结
一、项目背景
本人作为前端开发工程师，所在公司主营 VS Code 插件开发，本次需实现插件 Panel 内的日志输出展示功能，满足日志实时接收、页面渲染、本地持久化及高效交互的需求，同时保证插件长时间稳定运行。
二、核心业务功能
日志数据获取：后端通过windows的pipe通道实时推送日志数据，前端采用300ms 节流聚合数据，不做实时传输渲染，降低页面压力。
数据双端处理：节流后的日志数据，一方面写入 C 盘本地文件实现持久化，另一方面通过 VS Code 扩展进程与页面视图通信，传输至前端页面渲染。
日志展示模式
默认递增模式：日志数据持续追加，滚动条始终保持在底部，实时查看最新日志
自由浏览模式：用户手动调整滚动位置后，新增日志不会改变当前可视区域，保持浏览位置
数据管控规则：页面最多存储 20000 条日志，超出后执行首删尾插操作，始终保留最新数据。
附加交互功能：支持日志筛选（筛选基于全量 20000 条数据，而非当前筛选结果）、当前页面展示数据导出保存至文件。
三、核心技术选型
前端框架：React
虚拟列表组件：React-virtuoso 4.18.4，实现大数据量日志高效渲染
通信方式：VS Code 扩展进程与 Panel 页面 IPC 通信
性能优化：数据节流、虚拟滚动、数组原地操作优化
四、原有运行问题
日志功能持续运行（如每隔 300ms打印日志，连续运行一天一夜）后，出现页面白屏崩溃问题，通过浏览器控制台 Performance 工具排查，定位核心问题：
GC 频繁回收不及时：大量临时对象创建，导致垃圾回收机制超负荷，内存无法有效释放
JS 主线程运行过于频繁：高频数据更新、渲染及滚动操作，持续占用主线程
内存持续上涨：最终引发内存溢出，导致页面崩溃
五、已做优化操作
优化日志数组操作，等高频创建新对象的操
优化虚拟列表渲染逻辑，减少不必要的重渲染
优化滚动逻辑，避免高频调用滚动方法
logAllDataRef 是存储20000全量日志数据，方便筛选（筛选条件都是在全量数据中筛选）
logData 是我的虚拟列表展示的数据 data={logData}
